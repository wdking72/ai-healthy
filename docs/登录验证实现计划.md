# 登录验证实现计划

## 前置条件

项目已安装所需依赖（无需额外安装）：

| 依赖 | 用途 |
|------|------|
| `bcryptjs` | 密码加密与比对 |
| `jose` | JWT 签发与验证（兼容 Next.js Edge Runtime） |
| `mongodb` / `mongoose` | 数据库操作 |
| `next/headers` 中的 `cookies` | 设置 httpOnly Cookie（内置，无需安装） |

## 实现步骤

### □ 1. 添加密码比对工具函数

**文件：** `src/utils/psdHash.ts`

新增 `comparePassword` 函数，用于登录时比对用户输入的密码和数据库中存储的加密密码。

```ts
export const comparePassword = async (password: string, hashedPassword: string) => {
  return await bcrypt.compare(password, hashedPassword)
}
```

---

### □ 2. 新建 JWT 工具模块

**文件：** `src/lib/jwt.ts`

> JWT 全称 JSON Web Token，是一种用于在客户端和服务端之间安全传输信息的令牌标准。它由三部分组成：Header（头部）、Payload（载荷）和 Signature（签名）。

用 `jose` 实现 JWT 的签发和验证：

- **签发 Token（`signToken`）**
  - 接收 payload（包含 `userId`、`username`、`role`）
  - 用 `SignJWT` 生成 JWT
  - 设置过期时间（如 7 天）

- **验证 Token（`verifyToken`）**
  - 用 `jwtVerify` 验证 JWT 的签名和有效期
  - 返回 payload 或抛出异常

---

### □ 3. 添加 JWT 密钥到环境变量

**文件：** `.env.local`

```
JWT_SECRET=your-random-secret-key-here
```

> 密钥用于对 JWT 进行签名，确保 token 不被篡改。生产环境应使用足够长的随机字符串。

---

### □ 4. 完善登录 Server Action

**文件：** `src/lib/actions/auth.ts`

完善 `loginUser` 函数，逻辑如下：

```
用户提交登录表单
    ↓
1. 查询数据库 → 用户是否存在？
    ├── 否 → 返回 { success: false, message: '用户不存在' }
    └── 是 → 继续
2. bcrypt.compare() 比对密码
    ├── 错误 → 返回 { success: false, message: '密码错误' }
    └── 正确 → 继续
3. jose 签发 JWT（payload: userId, username, role）
4. cookies().set() 设置 httpOnly Cookie
5. 返回 { success: true, message: '登录成功' }
```

---

### □ 5. 完善登录页面

**文件：** `src/app/auth/login/page.tsx`

- 调用 `loginUser` Server Action
- 提交时按钮显示 loading 状态，防止重复提交
- 成功后用 `redirect('/')` 跳转到首页
- 失败用 `message.error()` 显示错误信息

---

### □ 6. 新增登录相关类型

**文件：** `src/lib/actions/type.ts`

新增 `LoginResult` 接口，统一登录接口的返回值格式。

## 验证流程总结

```
┌─────────────────────────────────────────────────┐
│                    用户输入                      │
│          用户名/邮箱 + 密码                      │
└────────────────────┬────────────────────────────┘
                     ↓
┌────────────────────┴────────────────────────────┐
│               loginUser Server Action             │
│                                                   │
│  1. DBconnect() ────────── 连接 MongoDB           │
│  2. User.findOne() ─────── 查找用户               │
│  3. comparePassword() ──── 比对密码               │
│  4. signToken() ────────── 签发 JWT               │
│  5. cookies().set() ────── 设置 httpOnly Cookie   │
└────────────────────┬────────────────────────────┘
                     ↓
┌────────────────────┴────────────────────────────┐
│              返回 { success, message }           │
│   成功 → 前端 redirect('/') 跳转首页             │
│   失败 → 前端 message.error() 显示提示            │
└─────────────────────────────────────────────────┘
```
